# frontend — SynapVox 웹 UI

담당: 도원

녹음본, 자료, 전사문, 그래프 뷰, AI 대화를 한 프로젝트 화면에서 다루는 React 프론트엔드입니다.

## 현재 흐름
- 홈에서 프로젝트를 만들고 프로젝트별 녹음본·자료를 관리
- 프로젝트 안에서 소스 카드, 그래프 뷰, AI 대화를 한 화면에 배치
- 웹 녹음 또는 녹음된 파일 업로드 후 `/api/stt/transcribe`로 전사 요청
- 전사 결과를 화자별 음성 기록으로 매핑하고 녹음본 상세 화면에서 확인·편집·복사

## 연동
- 개발 서버의 `/api` 요청은 `http://127.0.0.1:8000` 백엔드로 프록시합니다.
- 프로젝트 자료와 녹음본 참고 파일은 전사 요청 시 함께 전송할 수 있습니다.

## 소스 날짜(content_date)와 그래프 시간축
업로드 시 지정하는 날짜는 **자료가 만들어진 실제 시점**이지 적재한 날이 아닙니다. 이 값이
그래프의 시간축(`reference_time`)이 되어 Graphiti의 시간 추론(최신본/과거본 구분)에 쓰이므로,
경로별로 어떤 날짜가 붙는지 구분해 둡니다.

| 업로드 경로 | 그래프에 박히는 날짜 |
|---|---|
| 녹음 파일 업로드 | 사용자가 지정한 녹음 날짜 |
| 직접(웹) 녹음 | 제출 시점의 오늘 |
| **녹음에 첨부한 자료** | **그 녹음과 같은 날짜를 물려받음** (자료 따로 날짜를 묻지 않음) |
| 프로젝트 자료 단독 업로드 | 자료 날짜 모달에서 지정한 날짜, 미지정 시 오늘 |

> 첨부 자료가 녹음 날짜를 물려받는 것은 의도된 동작입니다. 날짜를 넘기지 않으면 백엔드가
> 적재 시각(오늘)으로 대체해 시간 추론이 왜곡되므로, 첨부 자료도 반드시 녹음 날짜를 전달합니다.

## 스택
Vite + React 19 + TypeScript

## 구조
- `src/App.tsx` — 주요 화면과 상태 관리
- `src/App.css` — 화면 레이아웃과 인터랙션 스타일
- `public/`

## 실행
```
cd frontend
npm install
npm run dev      # http://localhost:5173
npm run build
```
